# ✅ إصلاح مشكلة تسجيل الدخول 401

## 🎯 المشكلة

**الخطأ**: تسجيل الدخول يعيد 401 Unauthorized في الإنتاج
**المؤكد**: اختبار SQL في Supabase يعمل: `SELECT * FROM login_user('admin@sck.com', 'scq2025')`
**البيانات الصحيحة**: 
- البريد: `admin@sck.com`
- كلمة المرور: `scq2025`

---

## 🔍 السبب المحتمل

الكود كان **صحيح منطقياً**، لكن كانت هناك مشاكل محتملة:

1. **معامل action قد يحتوي على مسافات أو رموز خاصة**
   - مثل: `/api/auth?action=login%20\`
   - الحل: تنظيف المعامل

2. **طريقة استقبال البيانات من الطلب**
   - قد لا يتم تحليل JSON بشكل صحيح
   - الحل: إضافة سجلات للتحقق

3. **صيغة الرد من Supabase**
   - قد يعيد array أو object
   - الحل: التعامل مع الحالتين

4. **عدم وجود معلومات للتشخيص**
   - لا نعرف أين يفشل بالضبط
   - الحل: إضافة سجلات شاملة

---

## ✅ التعديلات على `api/auth.js`

### 1. التحقق من متغيرات البيئة
```javascript
if (!SUPABASE_URL || !SUPABASE_SERVICE_KEY) {
  console.error('Missing Supabase credentials')
}
```
**لماذا**: للتأكد من وجود الاتصال بـ Supabase

### 2. تنظيف معامل action
```javascript
const action = String(rawAction).trim().replace(/[\\/\s]/g, '').toLowerCase()
```
**لماذا**: إزالة المسافات والرموز الخاصة

### 3. سجلات الطلب
```javascript
console.log('Auth request:', {
  method: req.method,
  rawAction,
  normalizedAction: action,
  hasBody: !!req.body
})
```
**لماذا**: لنرى بالضبط ما يصل للـ API

### 4. دعم حقول متعددة
```javascript
const identifier = req.body.email || req.body.username || req.body.identifier
```
**لماذا**: الواجهة العربية تقول "البريد الإلكتروني أو اسم المستخدم"

### 5. سجلات محاولة تسجيل الدخول
```javascript
console.log('Login attempt:', {
  identifier,
  hasPassword: !!password
})
```
**لماذا**: لنرى البيانات المستلمة (بدون كشف كلمة المرور)

### 6. سجلات تفصيلية من Supabase
```javascript
console.log('Supabase RPC result:', {
  hasData: !!result,
  isArray: Array.isArray(result),
  dataLength: result ? result.length : 0,
  hasError: !!error
})
```
**لماذا**: لنرى بالضبط ما يعيده Supabase

### 7. التعامل مع صيغتي الرد
```javascript
let user = null
if (Array.isArray(result)) {
  user = result[0]
} else if (result && typeof result === 'object') {
  user = result
}
```
**لماذا**: Supabase قد يعيد array أو object

### 8. سجل المستخدم الموجود
```javascript
console.log('User found:', {
  id: user.id,
  email: user.email,
  role: user.role,
  is_active: user.is_active
})
```
**لماذا**: للتأكد من إيجاد المستخدم

### 9. سجل النجاح
```javascript
console.log('Login successful for:', user.email)
```
**لماذا**: للتأكد من اكتمال تسجيل الدخول

## 🚀 خطوات الاختبار

### 1. انتظر اكتمال النشر
- افتح: https://vercel.com/dashboard
- انتظر حتى يظهر "Ready"

### 2. اختبر بـ curl
```bash
curl -X POST https://sck-tawny.vercel.app/api/auth?action=login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@sck.com","password":"scq2025"}'
```

**الرد المتوقع عند النجاح**:
```json
{
  "success": true,
  "message": "Login successful",
  "token": "eyJhbGc...",
  "data": {
    "access_token": "eyJhbGc...",
    "user": {
      "email": "admin@sck.com",
      "full_name": "Super Admin",
      "role": "admin",
      "is_active": true
    }
  }
}
```

### 3. شوف Vercel Logs
1. روح Vercel Dashboard
2. اضغط على المشروع
3. روح تاب "Logs"
4. دور على السجلات

**المفروض تشوف**:
```
Auth request: { method: 'POST', normalizedAction: 'login', hasBody: true }
Login attempt: { identifier: 'admin@sck.com', hasPassword: true }
Calling Supabase RPC login_user
Supabase RPC result: { hasData: true, isArray: true, dataLength: 1 }
User found: { email: 'admin@sck.com', role: 'admin', is_active: true }
Login successful for: admin@sck.com
```

### 4. جرب في المتصفح
1. افتح: https://sck-tawny.vercel.app/login
2. افتح DevTools (F12)
3. سجل دخول:
   - البريد: `admin@sck.com`
   - كلمة المرور: `scq2025`
4. المفروض يوجهك للـ Dashboard
5. مفيش أخطاء 401 في Console

## 🔍 لو لسه في مشكلة

### شوف Vercel Logs وابحث عن:

#### لو شفت:
```
Login attempt: { identifier: undefined, hasPassword: false }
```
**المشكلة**: البيانات مش واصلة
**الحل**: تأكد من Content-Type: application/json

#### لو شفت:
```
Supabase RPC result: { hasError: true, errorMessage: 'function login_user does not exist' }
```
**المشكلة**: الـ function مش موجودة في Supabase
**الحل**: شغل `CREATE_LOGIN_FUNCTION.sql` في Supabase

#### لو شفت:
```
Supabase RPC result: { dataLength: 0 }
No user found - empty array
```
**المشكلة**: كلمة المرور غلط أو المستخدم مش موجود
**الحل**: تأكد في Supabase:
```sql
SELECT * FROM login_user('admin@sck.com', 'scq2025');
```

#### لو شفت:
```
Missing Supabase credentials: { hasUrl: false }
```
**المشكلة**: متغيرات البيئة مش موجودة
**الحل**: ضيفها في Vercel Dashboard → Settings → Environment Variables
