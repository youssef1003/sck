## 🔄 آلية الموافقة على الطلبات

### خطوات الموافقة:

1. **Client يقدم طلب:**
   - يسجل دخول
   - يروح `/careers`
   - يملأ النموذج ويقدم
   - الطلب يتحفظ بحالة `pending`

2. **Admin أو Sub-Admin يراجع:**
   - يدخل `/admin/careers`
   - يشوف كل الطلبات
   - يقدر يغير الحالة:
     - `pending` → `approved` (مقبول)
     - `pending` → `rejected` (مرفوض)

3. **Employer يشوف المقبولين:**
   - يدخل `/employer/dashboard`
   - يشوف الطلبات اللي حالتها `approved` فقط
   - يقدر يبحث ويشوف التفاصيل

## 🎯 حالات الاستخدام

### Scenario 1: زائر يتصفح المنصة
1. يدخل الموقع بدون تسجيل
2. يشوف الصفحة الرئيسية
3. يشوف الخدمات والاستشارات
4. يشوف صفحة التوظيف
5. لو حب يقدم → يطلب منه التسجيل

### Scenario 2: Client يقدم على وظيفة
1. يسجل من `/register` → يختار "عميل"
2. يسجل دخول → يروح `/dashboard`
3. يروح `/careers` → يقدم طلب
4. الطلب يتحفظ بحالة `pending`
5. يرجع `/dashboard` → يشوف طلبه "قيد المراجعة"

### Scenario 3: Sub-Admin يوافق على طلب
1. يدخل بـ `subadmin1` / `scq2025sub1`
2. يروح `/admin/dashboard`
3. يضغط على "التوظيف"
4. يشوف الطلبات قيد المراجعة
5. يغير الحالة من `pending` إلى `approved`

### Scenario 4: Employer يشوف المرشحين
1. يسجل من `/register` → يختار "صاحب عمل"
2. يسجل دخول → يروح `/employer/dashboard`
3. يشوف المرشحين المقبولين فقط
4. يبحث عن مرشح معين
5. يشوف تفاصيله الكاملة

### Scenario 5: Admin يدير كل شيء
1. يدخل بـ `admin` / `scq2025`
2. يروح `/admin/dashboard`
3. يشوف كل القوائم
4. يعدل أي محتوى
5. يدير الطلبات (قبول/رفض)
6. يدير كل شيء
