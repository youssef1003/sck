## 🔧 الخطوات المطلوبة من المستخدم

### الخطوة 1: تشغيل SQL Migration في Supabase

1. افتح **Supabase Dashboard**
2. اذهب إلى **SQL Editor**
3. انسخ والصق الكود التالي:

```sql
-- إضافة حقل رابط حجز الاستشارة لجدول المقالات
ALTER TABLE blog_posts 
ADD COLUMN IF NOT EXISTS booking_link TEXT;

-- تحديث المقالات الموجودة بالرابط الافتراضي
UPDATE blog_posts 
SET booking_link = '/consultation'
WHERE booking_link IS NULL;

-- التحقق من النتيجة
SELECT 
    'تم إضافة حقل booking_link بنجاح' as status,
    COUNT(*) as total_posts,
    COUNT(booking_link) as posts_with_link
FROM blog_posts
WHERE deleted_at IS NULL;
```

4. اضغط **Run** أو **F5**
5. تأكد من ظهور رسالة النجاح

### الخطوة 2: إعادة نشر الموقع على Vercel

بعد تشغيل SQL Migration، قم بإعادة نشر الموقع:

```bash
git add .
git commit -m "Add booking link feature to blog posts"
git push
```

أو من Vercel Dashboard:
- اذهب إلى **Deployments**
- اضغط **Redeploy**

## 📝 كيفية الاستخدام

### إضافة مقال جديد مع رابط حجز

1. اذهب إلى **Dashboard** → **إدارة المدونة**
2. اضغط **مقال جديد**
3. املأ البيانات:
   - العنوان
   - المقتطف
   - المحتوى
   - التصنيف
   - رابط الصورة
   - **رابط حجز الاستشارة** (مثال: `/consultation` أو `/contact`)
4. اضغط **نشر المقال**

### تعديل رابط الحجز لمقال موجود

1. اذهب إلى **Dashboard** → **إدارة المدونة**
2. اضغط أيقونة **تعديل** (✏️) على المقال
3. عدّل حقل **رابط حجز الاستشارة**
4. اضغط **تحديث المقال**

### حذف مقال

1. اذهب إلى **Dashboard** → **إدارة المدونة**
2. اضغط أيقونة **حذف** (🗑️) على المقال
3. أكد الحذف
